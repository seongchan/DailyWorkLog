# Daily Work Log

일일 마크다운 노트에서 작업 시간 기록(이벤트)과 할 일 목록을 바로 시각화해주는 옵시디언 플러그인입니다. YAML frontmatter가 필요 없습니다.

*[English README is here.](./README.md)*

## 주요 기능

- **사이드바 뷰**: 오른쪽 사이드바에 월간 캘린더, 이벤트(타임라인) 섹션, To-Do 섹션이 함께 표시됩니다.
- **인라인 편집**: 노트를 직접 열지 않고도 사이드바에서 바로 이벤트/할 일을 추가·수정·삭제할 수 있습니다.
- **최소한의 메타데이터**: 별도의 frontmatter 없이, 마크다운 본문의 헤더 3개만으로 동작합니다.
- **다국어 마커 인식**: 노트에 영어(`_Event` / `_ToDo` / `_Diary`) 또는 한국어(`_이벤트` / `_할일` / `_다이어리`) 헤더 중 무엇을 써도, 설정과 무관하게 항상 동시에 인식됩니다.
- **서로 독립적인 두 가지 테마**: 사이드바 전용 라이트/다크 설정(Obsidian 전체 테마와 별개)과, Obsidian 테마를 그대로 따르는 대시보드.
- **대시보드**: 볼트 전체 요약 — 총 노트 수, 연속 작성일(스트릭), 기록 기간, 최근 30일 To-Do 완료율 및 총 이벤트 기록 시간.
- **UI 언어**: 설정에서 영어/한국어 전환 가능.

## 일일 노트 형식

일일 노트 파일명은 `YYYY-MM-DD.md` 형식이어야 합니다. 플러그인으로 새 노트를 생성하면 다음과 같은 형태로 만들어집니다.

```markdown
# _Event

- 9:00 - 10:30 팀 회의

# _ToDo

- [x] 주간 보고서 제출
- [ ] PR 리뷰

# _Diary

자유롭게 메모하는 공간입니다 — 사이드바에는 노출되지 않습니다.
```

- 세 블록은 어떤 순서로 있어도 상관없습니다.
- 세 헤더가 하나도 없는 파일은, 파일 어디에 있든 `- [ ]`/`- [x]` 줄은 할 일로, `- HH:mm - HH:mm ...` 줄은 이벤트로 인식하는 방식으로 대체 동작합니다.

## 설치 방법

아직 커뮤니티 플러그인 브라우저에는 등록되어 있지 않습니다. 수동 설치 방법은 다음과 같습니다.

1. [릴리스](../../releases)에서 `main.js`, `manifest.json`, `styles.css` 세 파일을 내려받습니다 (또는 아래처럼 직접 빌드).
2. 세 파일을 `<볼트 경로>/.obsidian/plugins/daily-work-log/`에 복사합니다.
3. 옵시디언에서 **설정 → Community plugins**로 이동해 (필요하면 Restricted mode를 끄고) **Daily Work Log**를 활성화합니다.

### 소스에서 직접 빌드하기

```bash
npm install
npm run build   # dist/main.js, dist/manifest.json, dist/styles.css 생성
```

개발 중에는 `npm run dev`를 사용하세요 — 코드가 바뀔 때마다 자동으로 다시 빌드됩니다 (esbuild watch 모드).

## 사용 방법

- **대시보드 열기**: 리본 아이콘(calendar-clock)을 클릭하거나, 커맨드 팔레트에서 **"Daily Work Log 대시보드 열기"**를 실행하세요. 메인 워크스페이스의 일반 탭으로 열립니다.
- **사이드바 열기**: 커맨드 팔레트(`Cmd/Ctrl+P`)에서 **"Daily Work Log 사이드바 열기"**를 실행하세요. **설정 → 단축키**에서 원하는 단축키를 지정할 수도 있습니다.
- 캘린더에서 날짜를 클릭하면 해당 날짜의 노트가 열립니다 (파일이 없으면 확인 팝업이 뜹니다). **오늘** 버튼을 누르면 확인 없이 바로 오늘 노트로 이동합니다.

## 설정

- **마커 언어 (Marker language)**: 새 노트를 만들 때 어떤 언어의 헤더 라벨을 삽입할지 결정합니다. 이미 작성된 노트는 이 설정과 무관하게 어느 언어로 작성됐든 계속 정상적으로 인식됩니다.
- **사이드바 테마**: 사이드바에만 적용되는 라이트/다크 설정입니다.
